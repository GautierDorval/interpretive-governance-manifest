# Maximalist governance package

Status: proposed  
Normative language: SHOULD / MAY

## Purpose

A maximalist governance package declares the expected governance artifacts for a site or implementation that wants machine-readable interpretive governance to be discoverable, auditable, and non-minimal.

## Required categories

A complete package SHOULD declare artifacts for:

- public AI policy;
- identity and entity graph;
- canon and claims;
- source precedence;
- interpretation policy;
- response legitimacy;
- External Authority Control;
- Q-Layer or equivalent legitimacy rules;
- anti-plausibility constraints;
- CCL, if implemented;
- semantic-boundary, if implemented;
- semantic router;
- intent map;
- site coherence map;
- `llms.txt` and related discovery files;
- root and `.well-known` entrypoints;
- HTML and HTTP `Link` relations;
- package manifest;
- validation report or audit status.

## Boundary

A package manifest proves inventory, not behavioral compliance.
