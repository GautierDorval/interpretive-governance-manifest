Interpretation integrity audit protocol

Purpose

This protocol defines a reproducible method for producing an opposable audit report that characterizes divergence between a declared corpus snapshot, machine-generated interpretations, and a verifiable evidence chain.

It does not introduce new norms. It operationalizes existing interpretive governance constraints by requiring traceability and legitimate non-response when verification is not possible.

Informative label (fr-CA): Audit d’intégrité interprétative.

Governed by

R3, Anti-faux audit, Anti-identity inference.

Output artifact

  • Audit report JSON Schema
  • Minimal example report

Validity

Audit validity is conditional on the stability of audit variables (snapshot hash, boundary, QuerySet version, model identity, and declared run parameters). If a variable changes, the audit must be refreshed.

Non-objectives

  • No certification or compliance claim
  • No legal advice
  • No ranking of systems
  • No SEO performance guarantee
